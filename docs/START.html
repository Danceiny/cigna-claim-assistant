Cigna Claim Assistant

本地识别 PDF/图片，按服务日期一天一单，在已登录的 Cigna 页面提交。

使用前确认

  • 必须安装到你平时登录 Cigna 的 Chrome，不要用临时 Chrome profile。
  • 真实提交前先点扩展里的 检查 Cigna 页面 和 提交预检；需要验证真实页面流程时可先跑 真实页面彩排。
  • 日期由扩展在 Cigna 日期选择器里点击，不会手动输入。

安装

  1. macOS 上可先双击 OPEN_INSTALLER.command；它会打开本页、Chrome 扩展管理页，并把 extension 文件夹路径复制到剪贴板。
  2. 打开 Chrome 的 chrome://extensions。
  3. 打开右上角 Developer mode。
  4. 点击 Load unpacked。
  5. 选择当前解压目录里的 extension 文件夹。
  6. 在同一个 Chrome 里登录 Cigna Envoy。
  7. 可选：在 uTools 中导入 cigna-claim-assistant-utools.upx，作为可选择或拖入报销目录的桌面扫描和计划导出入口；uTools 可导出 Chrome 扩展可导入的设置备份。

产品形态

  • Chrome 插件负责复用已登录 Cigna 页面并执行真实提交。
  • uTools 插件负责桌面拖入目录、本地扫描、OCR/压缩计划和导出 Chrome 设置备份。
  • release zip 只是总交付包，不是第三种产品形态。

推荐流程

  1. 打开 Cigna 中文提交页，点击扩展图标。
  2. 确认被保险人、报销理由、治疗国家/地区、理赔类型、就诊类型和付款账户关键词；长期病症最早治疗日期和最早提交服务日期都可留空。
  3. 拖入报销 PDF 文件夹；同名 .txt OCR 文本可一起拖入，只用于识别，不会上传。
  4. 点击 识别文件，检查 将提交 / 将跳过 / 需压缩。
  5. 如果 需压缩 不为 0，先点击 压缩 PDF。
  6. 点击 提交预检，确认日期、字段、附件名、大小和 hash。
  7. 可选：点击 真实页面彩排，它会上传附件并走到最终检查页，但不会勾选免责声明、不会点击最终提交，也不会写入防重账本。
  8. 确认无误后点击 自动处理并提交。

防重防漏

  • 只有发票/收据会阻塞。
  • 只有医疗/理赔表但没有发票/收据会阻塞。
  • 无法识别服务日期会阻塞。
  • 同一文件 hash 会跳过。
  • 提交成功日期会进入本地账本。
  • 超 6 MB PDF 或单日总附件超限会先压缩。
